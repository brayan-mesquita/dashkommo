Guia Definitivo: Gemini CLI

Este manual cobre tudo, desde o básico até técnicas avançadas de engenharia assistida por IA.

1. Comandos de Barra (Slash Commands)

Os comandos de barra controlam a sessão sem afetar o código do projeto.

ComandoO que faz
/modelTroca o modelo (Pro, Flash, Lite, Auto).
/planEntra no Modo de Plano (estratégia antes da ação).
/rewindVolta no tempo (remove a última interação).
/resumeLista e recupera conversas anteriores.
/initCria um arquivo GEMINI.md no projeto.
/statsMostra custo, tokens e performance da sessão.
/compressResume a conversa para economizar memória e tokens.
/restoreDesfaz alterações feitas por uma ferramenta.
Atalho: Pressione Esc duas vezes para acionar o /rewind interativo.

2. Subagentes: Os Especialistas

O Gemini CLI não é apenas uma IA, é uma equipe. Você pode invocar especialistas:

  • @codebase_investigator: O melhor para mapear arquitetura e bugs complexos.
  • @cli_help: Sabe tudo sobre o próprio Gemini CLI.
  • @generalist: Excelente para tarefas de massa (refatorar 20 arquivos de uma vez).
# Exemplo de invocação direta:
@codebase_investigator encontre onde o token de autenticação é validado.

3. Domínio do Contexto (A Alma do Agente)

A IA decide o que fazer baseada em arquivos de instrução (Markdown).

A Hierarquia:

  1. Global (~/.gemini/GEMINI.md): Suas preferências de estilo e vida.
  2. Projeto (./GEMINI.md): Regras da empresa ou do repositório.
  3. Diretório (./src/api/GEMINI.md): Regras específicas para aquele módulo.
Memória Persistente: Use a ferramenta save_memory. O que você salvar nela será lembrado em TODAS as próximas sessões do projeto.

4. MCP: Conectando o Mundo

MCP (Model Context Protocol) permite que a IA use ferramentas de terceiros.

# Adicionando uma ferramenta (ex: busca em documentação)
gemini mcp add docs "npx -y mcp-server-documentation"

# Comandos de gestão
/mcp list          # Ver o que está rodando
/mcp reload docs   # Atualizar a ferramenta

5. Configurações e Variáveis

Configure seu terminal para máxima eficiência.

VariávelDescrição
GEMINI_API_KEYSua chave de acesso (essencial).
GEMINI_MODELDefine o modelo padrão (ex: pro).
GEMINI_SANDBOXDefine se comandos rodam em docker ou local.

6. O Fluxo de Trabalho Ideal

  1. Pesquisa: Use grep_search e glob para entender o terreno.
  2. Estratégia: Use /plan para desenhar a solução.
  3. Execução: Deixe a IA editar arquivos usando replace ou write_file.
  4. Validação: Peça para a IA rodar os testes: npm test ou pytest.
Segurança: Nunca peça para a IA imprimir arquivos .env ou segredos. O sistema possui redação automática, mas a cautela é sua responsabilidade.
